## Beyond the catalog

If you need a token that isn't listed above, follow this decision flow:

1. **Is the value semantic in nature?** (a colour role like "warning", a spacing role like "card-padding") → Add it to `semantic/` or `responsive/`.
2. **Is it raw/descriptive?** (a hex value, a specific px number) → Add it to `primitives/`.
3. **Is it scoped to one component?** → Add it to `components.tokens.json` — but FIRST confirm it earns its keep (see the three gates in `references/component-families.md`).

Never invent a token at the component tier if a semantic one would do. Re-read SKILL.md → "When does a component-tier token earn its keep?" before adding to `components.tokens.json`.
